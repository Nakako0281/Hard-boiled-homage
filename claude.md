# 開発ルール: タスクの進め方とテスト基準 📝
1. タスクの進め方
タスクは**ToDo.md**に記載されているものを、一つずつ順番に進めてください。

同時に複数のタスクに着手することは避けてください。

2. 実装と検証 (テスト)
一つのタスクの実装が完了したら、コミットする前に必ず以下のテストを実行し、実装が正しく機能することを確認してください。

Jest を用いて、ユニットテスト（コンポーネントや関数の単体テスト）を実行すること。

Playwright を用いて、E2Eテスト（ユーザー視点での一連の動作確認）を実行すること。

3. コミットの仕方
一つのToDoがテストまで終わり、エラーが出ないことを確認できたらコミットを作成しリモートにプッシュを行うこと。

4. エラーの対処法
必ず根本的な原因を考えて修正すること。決して対処療法を行ってはならない。
＊（例えばテストが通らないとき、テストコードを削除してエラーを無くす等のようなことは決してしてはならない。）
